# Patients Section Specification

## Overview

The Patients section provides comprehensive patient management capabilities including listing, viewing, creating, and editing patient records. It serves as the central hub for all patient-related data and interactions.

## Screens

### List View
**Path:** `/patients`

A searchable, filterable list of all patients in the system.

**Features:**
- Table view with sortable columns (Name, Email, Phone, DOB, Created)
- Grid/Card view for visual browsing
- Search by name, email, or phone
- Filter by date range, insurance provider
- Bulk actions (export, email)
- Pagination with customizable page size

### Patient Detail
**Path:** `/patients/:id`

Comprehensive view of a single patient's information.

**Features:**
- Personal information card
- Contact details
- Emergency contact
- Medical information (allergies, medications, history)
- Insurance details
- Appointment history
- Documents list
- Activity timeline

### New Patient Form
**Path:** `/patients/new`

Form to register a new patient.

**Fields:**
| Field | Type | Required | Validation |
|-------|------|----------|------------|
| First Name | text | Yes | 2-50 characters |
| Last Name | text | Yes | 2-50 characters |
| Email | email | Yes | Valid email format |
| Phone | tel | Yes | Valid phone format |
| Date of Birth | date | Yes | Must be in past |
| Gender | select | No | Male/Female/Other/Prefer not to say |
| Blood Type | select | No | A+/A-/B+/B-/O+/O-/AB+/AB- |
| Address | text | No | Max 200 characters |
| City | text | No | Max 100 characters |
| State | text | No | 2-letter code |
| ZIP Code | text | No | 5 or 9 digit format |
| Emergency Contact | text | No | Max 100 characters |
| Emergency Phone | tel | No | Valid phone format |
| Allergies | multi-select/tags | No | Free text tags |
| Current Medications | multi-select/tags | No | Free text tags |
| Medical History | textarea | No | Max 2000 characters |

### Edit Patient
**Path:** `/patients/:id/edit`

Edit existing patient information with same fields as new patient form.

## Data Model

```typescript
interface Patient {
  id: string;
  firstName: string;
  lastName: string;
  email: string;
  phone: string;
  dateOfBirth: string;
  gender?: "male" | "female" | "other" | "prefer_not_to_say";
  bloodType?: string;
  address?: string;
  city?: string;
  state?: string;
  zipCode?: string;
  emergencyContact?: string;
  emergencyPhone?: string;
  allergies: string[];
  medications: string[];
  medicalHistory?: string;
  insurance?: {
    provider: string;
    policyNumber: string;
    groupNumber?: string;
  };
  createdAt: string;
  updatedAt: string;
}
```
