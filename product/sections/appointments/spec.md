# Appointments Section Specification

## Overview

The Appointments section allows users to schedule, manage, and track patient appointments. It provides a comprehensive view of all appointments with filtering, sorting, and calendar navigation capabilities.

## Screens

### List View
**Path:** `/appointments`

A comprehensive list of all appointments with various viewing options.

**Features:**
- Table view with sortable columns (Patient, Date & Time, Type, Reason, Status)
- Grid/Card view for visual appointment browsing
- Date navigation (Previous/Next day, Today)
- Search by patient name or reason
- Filter by status (Scheduled, Confirmed, Completed, Cancelled, No-Show)
- Stats overview (Today's appointments, Confirmed, Pending, Cancelled)

### Calendar View
**Path:** `/appointments/calendar`

Visual calendar interface for appointment scheduling.

**Features:**
- Weekly and monthly calendar views
- Drag-and-drop appointment rescheduling
- Color-coded by appointment type
- Time slot availability indicators

### New Appointment
**Path:** `/appointments/new`

Form to create a new appointment.

**Fields:**
| Field | Type | Required | Validation |
|-------|------|----------|------------|
| Patient | select | Yes | Must select existing patient |
| Date | date | Yes | Must be future date |
| Time | time | Yes | Must be within business hours |
| Duration | select | Yes | 15, 30, 45, 60 minutes |
| Type | select | Yes | In-person, Video, Phone |
| Reason | text | Yes | Max 200 characters |
| Location | text | No | Only for in-person appointments |
| Notes | textarea | No | Max 500 characters |

### Appointment Detail
**Path:** `/appointments/:id`

Detailed view of a single appointment.

**Features:**
- Full appointment information
- Patient quick info card
- Action buttons (Confirm, Reschedule, Cancel)
- Appointment history/timeline
- Notes and attachments

## Data Model

```typescript
interface Appointment {
  id: string;
  patientId: string;
  patientName: string;
  patientAvatar?: string;
  date: string;      // ISO date string
  time: string;      // HH:mm format
  duration: number;  // in minutes
  type: "in-person" | "video" | "phone";
  status: "scheduled" | "confirmed" | "completed" | "cancelled" | "no-show";
  location?: string;
  reason: string;
  notes?: string;
  createdAt: string;
  updatedAt: string;
}
```
